// Core Innovations

ExecutorAgent 四项优化

01 / Confidence Filter
RAG 置信度过滤
计算检索文档数与期望文档数之比，置信度 < 0.4 时拒绝生成，转为主动追问用户，避免低质量检索导致的错误回答
threshold < 0.4 → 追问
02 / Patient Profile
患者画像跨轮记忆
升级 MemoryAgent，每 6 条消息用 LLM 自动提取病史、用药、过敏信息，采用合并策略保证历史信息不被覆盖
diseases · medications · allergies
03 / Auto Follow-up
模糊输入自动追问
信息不足时主动追问，每次只问一个最关键的问题，用猜测+确认方式降低老年人回答难度，避免生成泛泛回答
单问题追问策略
04 / Adaptive Length
回答长度自适应
根据问题长度和关键词判断复杂度，简单问候给 1-2 句简洁回答，复杂病情给结构化详细分析，适配老年人阅读习惯
简单 → 1-2句 / 复杂 → 结构化

// New Module

慢病管理模块

🗄️

SQLAlchemy 数据模型

单表统一存储五类指标，通过 record_type 字段区分，后续扩展新指标无需改表结构

🔌

RESTful API 接口

POST /chronic/record 记录指标，GET /chronic/records/{'{type}'} 查询历史，GET /chronic/summary 供 Agent 读取最新数据

📈

趋势可视化

基于 Recharts 渲染折线图，血压/血糖/心率附带警戒参考线，2 条以上数据自动显示趋势图

🤖

Agent 感知指标

Agent 回答时可调用 /chronic/summary 获取患者最新健康数据，结合实际数值给出个性化建议
